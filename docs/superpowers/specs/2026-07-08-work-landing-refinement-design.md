# 工作落地頁質感優化設計

## 目標

將現有 `work-landing` 優化成 AS娛樂首頁的原生延伸頁。保留黑金品牌、核心承諾、工作資訊與 LINE 轉換，不重新設計品牌，也不修改首頁。

成功標準：

- 使用者能立即辨認頁面屬於 AS娛樂
- 手機首屏能看到主標、核心承諾與主要 CTA
- 手機總長度由目前約 10300px 降至約 7500 至 8000px
- 酒店公關與美容師差異能快速比較
- 不新增無法證明的收入、心得或人數資料

## 設計方向

採用「首頁原生延伸版」。直接沿用首頁的絲絨光影、置中構圖、金白文字漸層、細金框、數字證明與四項承諾比例。落地頁仍然比首頁提供更多工作資訊，但不再使用大量相似卡片堆疊。

## 首屏

- 保留固定導覽列、AS Logo 與匿名詢問入口
- 背景沿用首頁的絲絨光影層次，不增加幾何線條或其他裝飾
- 信任標籤縮成首頁風格的小型標籤列
- 主標保持「先詢問再決定／不簽約也不壓薪水」
- 將第二行作為金白漸層視覺重點，縮小桌機版最大字級，避免壓迫
- 四項承諾改為首頁相同比例的細金框資訊列
- 主要 CTA 為「先匿名詢問」，次要 CTA 為頁內工作比較
- 手機首屏高度以 844px 至 932px 常見裝置能看見主要 CTA 為準

## 疑慮與信任

「妳擔心的事」由四張獨立卡片改成一個完整編號區塊。桌機版使用 2x2 分隔，手機版使用緊湊單欄，每項保留合約、薪資、隱私與拒絕權資訊。

「關於大頭」整合經驗數字與本人回覆說明，避免首屏、關於區與比較區重複表達同一組信任訊息。

## 職缺比較

- 桌機版維持酒店公關與美容師雙欄比較
- 手機版使用兩個等寬切換按鈕，一次顯示一種工作內容
- 預設顯示美容師，因目前廣告投放以美容師為主要方向
- 切換內容不重新載入頁面，且能使用鍵盤與螢幕閱讀器操作
- 每種工作保留薪資方式、排班、適合對象與發薪方式
- 薪資警示緊接比較區，保持低飽和紅色與短分隔線

## 頁面節奏

依序保留：

1. 首屏
2. 四項常見疑慮
3. 雙職缺比較
4. 不知道怎麼選的引導
5. 關於大頭與經驗數字
6. 誠實經紀與常見不透明做法比較
7. 適合對象
8. FAQ
9. 最終 CTA 與頁尾

縮短方式：

- 合併重複信任文字
- 手機職缺一次只顯示一種
- 減少相同高度的大型框線
- 標題與段落間距採用首頁比例
- 不刪除影響決策的重要資訊

## CTA 與連結

- 保留個人 LINE 為主要 CTA
- 保留官方 LINE、八大影片、官方 IG 四格入口
- 固定手機 LINE 按鈕沿用現有邏輯，只在首屏離開後出現
- 保留 `site-config.js` 統一官方 IG 網址
- 保留 Meta Pixel `PageView` 與個人 LINE 的 `Contact` 事件

## 視覺限制

- 色彩維持黑、白、灰、金，綠色只用於 LINE，紅色只用於警示
- 卡片圓角不超過 6px
- 不加入人物圖、漸層球體、紫色或新品牌元素
- 不使用大量陰影；光效集中於主標、金線與主要 CTA
- 動畫只使用 0.3 至 0.5 秒淡入與微光，支援 `prefers-reduced-motion`

## 技術範圍

- 修改 `work-landing.html`
- 修改 `work-landing.css`
- 不修改首頁與其他頁面
- 不新增框架或外部 JavaScript 套件
- 使用原生 HTML、CSS 與少量 JavaScript 完成職缺切換

## 驗證

- 桌機：1440x1000 與 1280x720
- 手機：430x932、390x844、375x812
- 驗證無水平捲動、文字不被切斷、CTA 置中且可點
- 驗證職缺切換、FAQ、固定 LINE 按鈕與 reduced-motion
- 驗證個人 LINE、官方 LINE、TikTok、Instagram 與首頁連結
- 驗證 Pixel 載入及 LINE 點擊事件仍存在
